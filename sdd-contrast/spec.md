# 회원가입 OTP — 요구사항 명세 (EARS)

> 3강 데모 **라운드 2(SDD)** 의 입력이다. 라운드 1(바이브)에서는 이 파일을 일부러 주지 않는다.
> 도메인·수치는 강의용 가상 스펙이다.

## 인터페이스 (채점기와의 계약)

`otp.py`에 `Otp` 클래스를 만들고 아래 메서드를 둔다. 시간은 실제 시계 대신 정수 t(초)로 주입한다.

| 멤버 | 동작 |
| --- | --- |
| `issue(email, t=0) -> code` | OTP 코드를 발급한다 |
| `verify(email, code, t=0) -> bool` | 코드가 맞고 유효하면 True |
| `signup(email, code, t=0) -> bool` | 검증을 통과하면 가입시키고 True |
| `created` | 가입된 사용자 모음 (멱등 채점에 사용) |

## 수용기준 (EARS)

- **AC-1 (정상)** — 발급한 코드로 검증하면 가입에 성공해야 한다.
- **AC-2 (만료)** — WHEN 코드 발급 후 TTL(300초)이 지나면, 시스템은 검증을 거부해야 한다.
- **AC-3 (잠금)** — WHEN 한 사용자가 5회 연속 틀리면, 시스템은 이후 정답을 넣어도 거부해야 한다(무차별 대입 차단).
- **AC-4 (멱등)** — WHEN 같은 사용자가 가입을 두 번 요청하면, 시스템은 계정을 1건만 생성해야 한다.

> 이 네 줄이 곧 합격/불합격 기준이다(`acceptance.py`로 코드화). 명세가 '무엇이 맞는가'를 정의하고, `grade.py`가 그것을 검증한다.
