# 비기능 수용기준 (NFR)

> 출처(00_sources): 요구사항정의서 §6 성능(PER)·§5 인터페이스(SIR)·§8 제약(CONR), 데이터 명세서 §5 품질 게이트.
> 각 비기능 요구를 측정 가능한 목표와 검증 위치로 정제한다. (SDD 1단계 '구조화' 산출물)

| 항목 | 목표 (근거) | 검증 위치 / 방법 |
| --- | --- | --- |
| 조회 응답시간 | 시세 통계 조회 **P99 ≤ 300ms** (PER-001) | analytics read model 분리(AC-5) + 집계 인덱스. 부하 테스트로 P99 측정 |
| 수집 처리량 | 시군구당 **≥ 1만 건/분** 수집·정규화 (PER-002) | `numOfRows=1000` 페이징으로 호출 수 최소화. ingestion 배치 처리율 측정 |
| 조회·수집 부하 격리 | 수집 배치 중에도 조회 응답시간 무열화 (PER-003) | **CQRS 읽기 분리** — write(거래원장)와 read(MarketStat) 경로 분리 |
| 동시성·트래픽 한도 | 외부 일일 한도 **10,000건(개발) 초과 금지** (PER-004, CONR-001) | 시군구·계약월 단위 순차/제한 동시 수집. 호출 카운트 가드 |
| 회복력 | 외부 API 장애 격리, 부분 수집 허용 (SIR-005, SFR-011) | **resilience4j** 재시도(3)·서킷브레이커·타임아웃(AC-2) |
| 멱등성 | 동일 구간 재수집 시 **중복 0** (DAR-003) | 거래 자연키 **유니크 제약** + upsert(AC-4) |
| 가용성 | 핵심 조회 경로 **99.9%**, 무중단 배포·롤백 (PER-005) | 디스커버리(Eureka) 다중 인스턴스 + 게이트웨이 헬스체크 |
| 데이터 품질 게이트 | `resultCode=000`만 적재, 필수항목·금액 양수·면적 양수 (데이터 명세 §5) | 적재 전 검증 — 위반 건 스킵·보고(부분 수집), 전량 수집 보장 |

## 검증 전제 (멱등·결정성)
- 단위 검증(금액 변환·해제 판정·중위 집계)은 외부 data.go.kr 호출 없이 **순수 도메인 로직**으로 결정적이어야 한다.
- 인증키·네트워크는 런타임 전용이며, 비기능 게이트는 네트워크 없이 재현 가능해야 한다.

> 보안 관련 비기능(인증키 비노출)은 `09_security`를 정본으로 한다.
