# 보안 계획 (Security Planning)

> 출처(00_sources): 요구사항정의서 §7 보안(SECR)·§8 제약(CONR), API 공개명세 §7 트래픽·키 정책, 데이터 명세서.
> 외부 공개 API(data.go.kr) 연계 시의 보안 포스처를 정제한다. (SDD 1단계 '구조화' 산출물)

## 1. 인증키 비노출 (SECR-001 / SECR-003)
- data.go.kr `serviceKey`는 개인 발급 자격증명이다. **소스코드·컨테이너 이미지·설정 파일·git에 평문 저장을 금지**한다.
- 주입 경로는 환경변수 **`MOLIT_SERVICE_KEY`** 하나로 통일한다.
- 로그·트레이스·에러 응답에 인증키가 출력되지 않도록 **마스킹**한다. (URL 조립 시 쿼리스트링 노출 주의)

## 2. 설정 외부화 (SECR-002 / SECR-005)
- 엔드포인트·회복력 정책·인증키 참조는 **Config Server**로 외부화한다.
- 비밀값은 Config에 직접 두지 않고 `${MOLIT_SERVICE_KEY:}` 형태로 **환경변수 참조만** 한다.
- 키 만료·교체 시 **무중단 설정 갱신**으로 반영 가능해야 한다.

## 3. 데이터 보호 / 개인정보 (SECR-004)
- 본 API는 거래 당사자 식별정보를 포함하지 않는 공개 데이터다.
- 동(棟)·매도자/매수자 구분(`slerGbn`/`buyerGbn`)은 **비식별 범주값**이며, 동 정보는 등기 완료 건에 한해 제한 공개된다.
- 적재는 표준 스키마 **AptTransaction** 항목으로 한정해, 수집 경로에서 추가 개인정보가 섞이지 않도록 한다.

## 4. 감사·추적
- 수집 트리거(시군구·계약월·요청자)와 적재/스킵 건수를 감사 로그로 남긴다.
- 외부 호출 실패·서킷 개방·트래픽 한도 임박 이벤트를 운영 지표로 노출한다.

## 5. 위협 · 통제 요약
| 위협 | 통제 |
| --- | --- |
| 인증키 유출 | 환경변수 주입, git/이미지 제외, 로그 마스킹, 키 로테이션 |
| 외부 API 장애 전파 | 서킷브레이커 격리, 부분 수집 허용, 빈 결과 폴백 |
| 트래픽 한도 초과(차단) | 호출 카운트 가드, 백오프, 익일 재개 |
| 중복·변조 적재 | 자연키 멱등 upsert, 표준 스키마·품질 게이트 검증 |
