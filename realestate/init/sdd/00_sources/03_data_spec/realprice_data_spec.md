# 데이터 명세서: 아파트 매매 실거래가 원천 데이터

> 문서 구분: 발주 산출물(데이터 명세서, Data Specification). SDD 0단계(00_sources) 입력 baseline입니다.
> 이 문서는 외부 원천 데이터의 항목 정의, 코드 도메인, 표기 규칙, 결측 규칙, 품질 기준을 박제합니다.
> API 호출 계약 정본은 `00_sources/01_apis/molit_apt_trade_api.md`, 요구사항 정본은 `00_sources/02_requirements/realfield-부동산실거래.md`입니다.
> 내부 표준 스키마(AptTransaction)로의 사상(map)은 학습자가 `01_planning/04_data`에서 수행합니다. 본 문서는 그 사상의 단일 사실 원천입니다.

## 2. 코드 도메인 (Code Domain)

> 원천이 코드값 또는 한정된 범주값으로 내려오는 항목입니다. 시스템은 범주값을 변형하지 않고 원천 표기를 보존하되, 적재·집계 판정에 쓰는 항목(특히 해제여부)은 아래 도메인으로 해석합니다.

### 2.1 거래유형 (dealingGbn)

| 값 | 의미 |
| --- | --- |
| 중개거래 | 공인중개사를 통한 거래 |
| 직거래 | 당사자 간 직접 거래 |
| (공백) | 구분 정보 미제공 |

### 2.2 매도자·매수자 구분 (slerGbn / buyerGbn)

원천은 거래 주체를 비식별 범주값으로 제공합니다. 개인 식별정보는 포함하지 않습니다(SECR-004). 관측되는 주요 범주값은 다음과 같으며, 시스템은 원천 표기를 그대로 보존합니다.

| 값(예) | 의미 |
| --- | --- |
| 개인 | 자연인 |
| 법인 | 법인 사업자 |
| 공공기관 | 국가·지자체·공공기관 |
| 기타 | 위로 분류되지 않는 주체 |
| (공백) | 구분 정보 미제공 |

### 2.3 토지임대부 아파트 여부 (landLeaseholdGbn)

| --- | --- |

| --- | --- |
| 1 | 일반 지번(대지) |

| 값 | 의미 |
| --- | --- |

> 시세 집계 제외 판정의 근거 항목입니다(AC-3, DAR-004). 시스템은 이 도메인을 해석해 `canceled` 플래그로 저장합니다.

## 3. 표기 규칙 (Notation Rule)

### 3.1 거래금액 (dealAmount)

- 단위는 만원입니다. 천 단위 콤마와 선행 공백을 포함할 수 있습니다. 예: `  82,500`.
- 시스템은 공백·콤마를 제거해 만원 정수를 얻고, 원 단위로 환산하기 위해 10,000을 곱합니다. 예: `  82,500` → `825,000,000`(원).
- 정규화 규칙 정본은 `01_planning/04_data`의 금액 정규화 절(DAR-002)입니다.

### 3.2 날짜

- 계약일은 `dealYear`·`dealMonth`·`dealDay` 세 정수 항목으로 분리 제공됩니다. 시스템은 세 항목으로 계약일자를 구성합니다.
- 등기일자(`rgstDate`)와 해제사유발생일(`cdealDay`)은 `YY.MM.DD` 두 자리 연도 표기입니다. 예: `24.07.10`. 시스템은 이를 4자리 연도 날짜로 변환하며, 공백은 결측(null)으로 처리합니다.

### 3.3 면적

- 전용면적(`excluUseAr`)은 ㎡ 단위 실수입니다. 소수 둘째 자리까지 제공되는 경우가 일반적입니다. 예: `84.97`.
- ㎡당 단가는 거래별로 `거래금액(원) / 전용면적(㎡)`으로 계산하며, 집계는 그 분포의 중위값을 사용합니다(DAR-007).

### 3.4 법정동코드 (LAWD_CD 체계)

- 행정표준코드관리시스템의 법정동코드는 10자리입니다. 구성은 시도(2자리) + 시군구(3자리) + 읍면동(3자리) + 동·리(2자리)입니다.
- 수집 요청 변수 LAWD_CD는 앞 5자리(시도 2 + 시군구 3)를 사용합니다. 예: `11110`은 서울특별시(11) 종로구(110)를 의미합니다.
- 본 사업은 전국 250개 시군구 코드를 마스터로 보관하고, 매월 (시군구 × 계약월) 조합으로 수집을 구동합니다(DAR-008).

## 5. 데이터 품질 기준 (Quality Gate)

> 수집 서비스가 원천 응답을 표준 원장으로 적재하기 전에 통과시켜야 하는 최소 품질 기준입니다.

| 기준 | 내용 | 처리 |
| --- | --- | --- |
| 응답 정상 | 헤더 `resultCode = 000`인 응답만 적재 대상입니다. | 비정상 응답은 적재하지 않고 회복력 정책으로 처리(SIR-005). |
| 필수 항목 충족 | `sggCd`·`umdNm`·`aptNm`·`excluUseAr`·`floor`·`dealYear`·`dealMonth`·`dealDay`·`dealAmount`가 모두 유효해야 합니다. | 필수 항목 결측·파싱 실패 건은 스킵하고 부분 수집을 허용(SFR-011). |
| 금액 유효 | 거래금액이 정상 변환되어 원 단위 양수여야 합니다. | 변환 실패 건은 스킵·보고. |
| 면적 유효 | 전용면적이 0보다 커야 합니다. | 위반 건은 스킵·보고. |
| 멱등 보장 | 동일 자연키 재수집 시 중복 행을 만들지 않습니다. | 자연키 기반 upsert(DAR-003, AC-4). |
| 전량 수집 | `totalCount` 대비 누락 없이 전 페이지를 수집해야 합니다. | 페이징 종료 판정으로 보장(SFR-002). |

## 6. 원천 → 내부 표준 사상 요약 (참고)

> 사상의 정본은 `01_planning/04_data`입니다. 본 절은 데이터 명세 관점의 요약입니다.

| 원천 항목 | 내부 표준(AptTransaction) | 핵심 변환 |
| --- | --- | --- |
| sggCd | sggCd | 동일 |
| umdNm / aptNm | umdNm / aptNm | trim |
| excluUseAr | exclusiveArea | 문자열 → double |
| floor / buildYear | floor / buildYear | 문자열 → int(건축년도 결측 허용) |
| dealYear / dealMonth / dealDay | dealYear / dealMonth / dealDay | 문자열 → int |
| dealAmount | dealAmountWon | 콤마·공백 제거 → 만원 정수 → ×10000 |
| cdealType | canceled | `== "O"` → boolean |
| cdealDay | canceledDate | `YY.MM.DD` → date(결측 허용) |
| rgstDate | rgstDate | `YY.MM.DD` → date(결측 허용) |
| dealingGbn / slerGbn / buyerGbn / estateAgentSggNm / aptDong / landLeaseholdGbn | 동명 보존 항목 | trim, 결측 허용 |

> 정합의 핵심 변환은 네 가지입니다. 금액 만원에서 원, 문자열에서 숫자, 해제 코드에서 boolean, `YY.MM.DD`에서 날짜입니다. 이 네 변환을 공유 라이브러리 한곳에서 강제해 수집·분석의 집계 편차를 제거합니다.
