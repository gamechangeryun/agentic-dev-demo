# 비기능 수용기준 (NFR)

> 1단계 '구조화' 산출물입니다. 비기능 요구를 검증 위치와 함께 정리합니다.

| 항목 | 목표 | 검증 위치 |
| --- | --- | --- |
| 조회 성능 | 시세 통계 조회 P99 300ms | analytics read model 분리(AC-5) |
| 수집 처리량 | 시군구당 1만건/분 | ingestion 배치 + numOfRows 1000 페이징 |
| 회복력 | 외부 API 장애 격리, 부분 수집 허용 | resilience4j 재시도·서킷(AC-2) |
| 멱등성 | 재수집 중복 0 | 자연키 유니크 제약(AC-4) |
| 보안 | 인증키 비노출 | Config 외부화 + 환경변수(`MOLIT_SERVICE_KEY`) |
| 가용성 | 99.9%, 무중단 배포·롤백 | 디스커버리 + 게이트웨이 헬스체크 |

> 인증키는 코드·이미지·Config 파일에 평문으로 두지 않습니다. 환경변수로만 주입합니다.
