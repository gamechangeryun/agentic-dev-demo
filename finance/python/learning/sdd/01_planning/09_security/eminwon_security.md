# 전자민원 발급 · 보안·프라이버시 (계획)

- **동의 우선**: 활성 동의 없으면 어떤 처리도 시작하지 않는다 (AC-1 게이트).
- **동의 철회·파기**: 철회 시 수집 데이터 처리 중단·파기 + 동의 원장 append (AC-5).
- **가명처리(계획)**: 분석·상담 질의는 원본 식별자 대신 가명 키로만 응답: 본 데모는
  사용자 식별자를 불투명 문자열(`u1`, `u2`)로만 다뤄 PII를 포함하지 않는다.
- **근거 인용 가드레일**: 자격 미달 시 답을 지어내지 않고 발급을 거부한다 (AC-3).
- **감사 추적**: 동의 원장은 append-only(`server/shared/consent_ledger.py`)로 부여·철회
  이력이 누적된다.

> A·B조직 동석 환경을 고려해 실재 기관·개인정보·실명을 일절 포함하지 않는다.
