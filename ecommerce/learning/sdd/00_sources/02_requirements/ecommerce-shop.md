# 이커머스 쇼핑 서비스 요구사항 원문

> 이 문서는 데모의 유일한 입력입니다. 아래 문장 하나하나가 EARS 명세의 AC 한 줄,
> 테스트 한 케이스로 이어집니다. 모든 구현은 이 원문에서 출발합니다.

## 도메인 개요

쇼핑 서비스는 상품을 등록하고, 고객이 장바구니에 담아 주문하고, 결제하고,
배송 이행까지 가는 전 과정을 다룹니다. 하나의 배포 단위(모놀리식) 안에서 여섯 개의
bounded context 가 협력합니다. catalog 는 상품을, inventory 는 재고 예약을,
cart 는 장바구니를, ordering 은 주문 상태를, payment 는 결제와 환불을,
checkout 은 이들을 묶는 흐름을 담당합니다.

## 상품 (catalog)

- 상품을 등록하면 ACTIVE 상태로 생성됩니다. 가격이 0원 이하이면 생성 단계에서 즉시 거부됩니다.
- 상품 목록은 이름으로 검색하고 상태로 거를 수 있으며 페이지 단위로 조회됩니다.
- 재고를 더하거나 뺄 수 있습니다. 보유 재고보다 많은 수량을 빼려고 하면 거부됩니다.
- 상품을 아카이브하면 ARCHIVED 상태가 됩니다. 이후 모든 재고 변경이 거부됩니다.
- 동일한 멱등 키로 재요청하면 기존 상품을 그대로 반환합니다. 중복 생성은 방지됩니다.

## 재고 예약 (inventory)

- 주문이 들어오면 가용 재고를 먼저 예약으로 묶어 둡니다. 가용 재고는 물리 재고에서 예약 합계를 뺀 값입니다.
- 결제가 확정되면 예약을 확정하여 물리 재고를 실제로 차감합니다.
- 주문이 취소되면 예약을 해제하여 가용 재고를 되돌립니다.
- 동시에 들어온 두 예약의 합이 가용 재고를 넘으면 뒤 예약을 거부합니다. 재고를 초과 판매하지 않습니다.

## 장바구니 (cart)

- 장바구니를 만들고 상품을 담습니다. 같은 상품을 다시 담으면 수량이 합쳐집니다.
- 수량을 바꾸거나 항목을 빼거나 장바구니를 비울 수 있습니다. 수량을 0으로 바꾸면 항목이 제거됩니다.
- ARCHIVED 상품은 장바구니에 담을 수 없습니다.

## 체크아웃과 주문 (checkout · ordering)

- 체크아웃은 장바구니를 주문으로 바꿉니다. 각 줄의 재고를 예약하고, 모두 성공하면 주문을 생성합니다.
- 한 줄이라도 재고가 부족하면 이미 잡은 예약을 모두 풀고 체크아웃을 거부합니다. 부분 예약을 남기지 않습니다.
- 빈 장바구니로는 체크아웃할 수 없습니다. 주문 총액이 0원 이하이면 생성 단계에서 거부됩니다.
- 주문은 CREATED 로 시작해 결제 시 PAID, 이행 시 FULFILLED 로 한 방향 전환합니다.
- CREATED 또는 PAID 상태에서만 취소할 수 있습니다. FULFILLED 주문은 취소할 수 없습니다.
- 동일한 멱등 키로 체크아웃을 재요청하면 기존 주문을 그대로 반환합니다. 중복 주문은 방지됩니다.

## 결제와 환불 (payment)

- CREATED 주문만 결제할 수 있습니다. 이미 결제된 주문을 다시 결제하려 하면 거부됩니다.
- 결제가 승인되면 주문이 PAID 로 전환되고, 주문에 묶인 재고 예약이 확정됩니다.
- 결제가 거절되면 주문은 CREATED 로 남아 재시도하거나 취소할 수 있습니다.
- 동일한 멱등 키로 결제를 재요청하면 기존 결제를 그대로 반환합니다. 이중 청구는 방지됩니다.
- 결제된 주문을 취소하면 결제가 환불됩니다. 환불은 캡처된 결제에 대해서만 허용됩니다.
